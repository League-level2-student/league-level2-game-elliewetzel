Pong

by: Ellie Wetzel

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because my sister and I used to play Pong a lot when we were younger.

2. What I Learned

Users said to move the paddles faster to keep the other player from getting a point. As the score increases the speed of the ball also increases, but users suggested that I should make it move even faster. Additionally, Paddle 1 was originally controlled by the arrow keys which gave the player an advantage over the other, so I had to switch which keys controlled it.

3. Architecture of My Game

The classes are structured similar to League Invaders. I used a class to create the different panels with text, code each of the individual objects, draw the objects, and check the collisions.

4. Technical Implementation

In order for my game to work, I used timers to create delays, algorithms to check collisions and how they effect the ball, and key listeners to control how the paddles move. The particular aspect I'm proud of is the code that controls how the ball bounces off the paddles because I had to specifically code the spot where the ball hit the paddle and where it was directed.

5. Most Challenging Part

The most challenging part of this project was trying to eliminate the bug that let the ball pass through the right paddle. In order to fix this, I created a delay when the ball collided with each paddle to keep it from repeatedly switching direction.

6. Future Fixes and Updates

If I kept working on this project, I would improve the graphics. Also, I would make the game a little faster and add a few more challenges to make it harder and more entertaining for the players.
